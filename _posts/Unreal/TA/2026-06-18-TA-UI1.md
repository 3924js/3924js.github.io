## 밋밋하지 않은 UI만들기
1. 기획서 만들기
	* AI에 컨셉만 던져주고 구체화/문서화 시켜 달라고 하는 것도 방법.
2. 기획서 토대로 시안 만들어달라고 하기
	* 안좋은 예시! - 뽑아준 시안 이미지 그대로 UMG에 때려넣기
		* 창 사이즈 바뀔때마다 비율/배치 깨짐
		* 애니메이션 넣기 불가능
		* 수정/관리 어려움
3. 좋은 예시! - **버튼이랑 요소들 따로 뽑아서 배치**!
	* 만들어진 시안 바탕으로 각 UI 요소들 개별 이미지로 생성
	* 앵커 위치 잡고 배치하기
		* A - 타이틀만 버튼들과 따로 해서 중앙 정렬, 배경 이미지는 조금 늘어나도 보기 괜찮음.
		* B - 이미지 고정사이즈로 넣고, 테두리에 필러박스로 가려버리기 -> 비율 깨지지 않고 유지 가능
	* 크기/비율 변경할 때 스케일은 쓰지말기! - 자식에도 영향줌!
		* 대신 **사이즈에 *n 하면 곱한 값**으로 반영!
	* 버튼을 화면 전체에 깔고 시작 기능과 연결
		* 버튼 알파 0으로 해서 안보이게
	* 텍스트 시안성 개선!
		* 아웃라인 - 좋긴한데 촌스러울 수 있음
		* 쉐이더 - 기본 쉐이더는 느낌이 뭔가 이상함.
		* **텍스트만 복사해서 2개**로 사용, 하나를 그림자/아웃라이너처럼 자유롭게 연출 가능.
	* 애니메이션 없으면 밋밋함.
		* 페이드인, 움직임등 **위젯 애니메이션으로  살아있는 느낌** 넣기
	* 여기까지 해야 "최소한"의 볼만한 UI, 추가적인 작업들도 필요.
		* 

## 언리얼 신규기능 살펴보기
1. 메가라이트
	* 루멘 자체가 연산이 무거움! -> 가벼운 게임 만들거면 루멘대신 스크린 스페이스가 더 유리
	* 나나이트도 마찬가지! -> 로우폴리면 안쓰는게 더 저렴 
2. MCP 활용
	1. 플러그인 설치
		* plugin - Unreal MCP 추가, 터미널은 옵션(에디터에서 바로 킬수는 있는데, 한글 입력 불가)
	2. 재시작 후 설정 체크
		* MCP - Auto start server
	3. 커멘드 창에서 실행
		* ModelContextProtocol.GenerateClientConfig All
		* 후에 CLI가서 MCP 연결
		* 외부에서 하기 귀찮으면 언리얼 터미널 쓸 수 있지만, 영어만 사용 가능.
